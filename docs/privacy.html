Privacy Policy

Last updated: March 2026

Mail Cleanup ("the Application") is an open-source desktop tool that helps users bulk-manage their email inboxes. This policy explains what data is accessed, how it is used, and what leaves your device.

1. Data we access

When you connect Gmail or Outlook, the Application accesses the following information from your mailbox:

  • Sender name and email address
  • Email subject lines
  • Email message IDs (used internally to delete or move messages)
  • Folder/label structure (used to route archived emails to named folders)
  • The List-Unsubscribe header of individual emails (used for the unsubscribe feature)

Email bodies are never read or accessed.

2. Data stored on your device

The following files are created locally on your machine in the folder where the Application is run:

  • tokens/ — OAuth access and refresh tokens for Gmail and Outlook. These allow the Application to re-authenticate without requiring you to log in each time.
  • checkpoint.json — A resumable session state, deleted at the end of each session.
  • whitelist.json — A list of senders you have chosen to always keep.
  • rules.json — A list of saved per-sender decisions.
  • reports/ — Optional JSON session reports, created only when the --report flag is used.

None of these files are transmitted anywhere. They remain on your device and can be deleted at any time.

3. Data sent to third parties

If you configure an Anthropic API key (ANTHROPIC_API_KEY), the Application sends the following information to the Anthropic API for AI-powered email classification:

  • Sender name and email address
  • Email subject lines (up to 5 per sender)
  • Email count per sender

No email bodies, no personal identifiers beyond the above, and no OAuth tokens are ever sent to Anthropic.

If no API key is configured, no data is sent to any third party.

Email provider APIs (Google and Microsoft) receive standard OAuth API requests as required to list, move, and delete emails on your behalf.

4. OAuth scopes

Gmail: The Application requests the https://mail.google.com/ scope, which grants full read and write access to your Gmail account. This is required to permanently delete emails. No data beyond what is described above is accessed.

Outlook: The Application requests Mail.Read and Mail.ReadWrite delegated permissions via Microsoft Graph, which allow reading and modifying mail on your behalf.

5. Data retention

The Application does not operate any servers and retains no data on your behalf. All data described above exists only on your local machine. You can remove it at any time by deleting the relevant files.

You can revoke the Application's access to your email account at any time:

  • Gmail: myaccount.google.com/permissions
  • Outlook: account.live.com/consent/Manage

6. Children's privacy

The Application is not directed at children under 13 and does not knowingly collect information from children.

7. Changes to this policy

We may update this policy from time to time. The latest version is always available at this URL. Continued use of the Application after changes constitutes acceptance of the updated policy.

8. Contact

Questions or concerns about this privacy policy can be raised by opening an issue on the GitHub repository.
